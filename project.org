** Arquitectura de Componentes

*** PatrГіn de Routing
El proyecto utiliza un sistema de *hash-based routing* implementado manualmente en =App.tsx= sin necesidad de libraries como React Router. El estado =currentView= determina quГ© componente renderizar basado en el hash de la URL:

- =#home= - Homepage
- =#blog= / =#blog/:id= - Blog y artГӯculos
- =#projects= / =#projects/:id= - Proyectos
- =#about= / =#about/member/:id= - Nosotros y miembros
- =#events= / =#events/:id= - Eventos
- =#courses= / =#courses/:id= - Cursos
- =#publications= - Publicaciones
- =#contact= - Scroll al footer

*** Providers de Estado
- *ThemeProvider*: IntegraciГіn con =next-themes= para soporte de temas oscuro/claro. Persiste la preferencia en localStorage.
- *LanguageProvider*: ImplementaciГіn custom de i18n con soporte para =es= y =en=. Provee =useLanguage()= hook y contexto para todos los componentes.

*** Arquitectura de Datos
Los datos estГЎn almacenados como objetos TypeScript en =src/data/= con interfaces tipadas:

- *Projects*: Arreglo de objetos Project con soporte multiidioma en todos los campos de texto
- *Members*: Miembros categorizados (academic, student, formerStudents) con informaciГіn de contacto
- *Events/Courses*: Eventos con fechas, descripciones y metadatos
- *BlogPosts*: Generado dinГЎmicamente desde archivos Markdown

** Sistema de Blog con Markdown

El sistema de blog utiliza un pipeline de build custom:

1. *Contenido*: Archivos Markdown en =content/blog/[slug]/= con frontmatter YAML
2. *GeneraciГіn*: =scripts/build-blog.js= parsea directorios y genera =src/data/blogPosts.ts=
3. *Multilenguaje*: Detecta archivos =index.es.md= e =index.en.md= para cada post
4. *ImГЎgenes*: Copia automГЎticamente imГЎgenes del directorio del post a =public/blog-images/[slug]/=
5. *Renderizado*: ReactMarkdown con plugins para:
   - GFM (GitHub Flavored Markdown)
   - Math (KaTeX para fГіrmulas)
   - Syntax highlighting (react-syntax-highlighter)
   - Custom image handling

** Pipeline de Build

*** Build Script (=npm run build=)
#+BEGIN_SRC bash

#+END_SRC

1. Ejecuta =build-blog.js= para generar =blogPosts.ts=
2. Vite compila el proyecto a =build/=

5. MinificaciГіn con esbuild

*** Contenedor Docker

1. *Stage builder*: Node 20 Alpine, instala dependencias, ejecuta build
2. *Stage runtime*: Nginx Alpine sirve archivos estГЎticos desde =build/=
3. ConfiguraciГіn custom de Nginx en =container/nginx.conf=
