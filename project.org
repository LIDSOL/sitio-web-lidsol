** Stack Tecnológico

El sitio web de LIDSoL está construido sobre una arquitectura moderna de aplicaciones web de una sola página (SPA) utilizando las siguientes tecnologías principales:

- *Framework frontend*: React 18.3.1 con TypeScript para tipado estático
- *Build tool*: Vite 6.4.1 para desarrollo rápido y construcción optimizada
- *Estilos*: Tailwind CSS 4.x con plugin @tailwindcss/typography para contenido markdown
- *Componentes UI*: Radix UI (suite completa de primitivas accesibles) + shadcn/ui pattern
- *Iconos*: Lucide React
- *Markdown*: react-markdown con remark-gfm, remark-math, rehype-katex, react-syntax-highlighter
- *Gráficos*: Recharts para visualizaciones de datos
- *Formularios*: React Hook Form
- *Animaciones*: Motion (anteriormente Framer Motion)
- *Procesamiento de imágenes*: Sharp + vite-plugin-image-optimizer
- *SVG*: SVGO para optimización

** Arquitectura de Componentes

*** Patrón de Routing
El proyecto utiliza un sistema de *hash-based routing* implementado manualmente en =App.tsx= sin necesidad de libraries como React Router. El estado =currentView= determina qué componente renderizar basado en el hash de la URL:

- =#home= - Homepage
- =#blog= / =#blog/:id= - Blog y artículos
- =#projects= / =#projects/:id= - Proyectos
- =#about= / =#about/member/:id= - Nosotros y miembros
- =#events= / =#events/:id= - Eventos
- =#courses= / =#courses/:id= - Cursos
- =#publications= - Publicaciones
- =#contact= - Scroll al footer

*** Providers de Estado
- *ThemeProvider*: Integración con =next-themes= para soporte de temas oscuro/claro. Persiste la preferencia en localStorage.
- *LanguageProvider*: Implementación custom de i18n con soporte para =es= y =en=. Provee =useLanguage()= hook y contexto para todos los componentes.

*** Arquitectura de Datos
Los datos están almacenados como objetos TypeScript en =src/data/= con interfaces tipadas:

- *Projects*: Arreglo de objetos Project con soporte multiidioma en todos los campos de texto
- *Members*: Miembros categorizados (academic, student, formerStudents) con información de contacto
- *Events/Courses*: Eventos con fechas, descripciones y metadatos
- *BlogPosts*: Generado dinámicamente desde archivos Markdown

** Sistema de Blog con Markdown

El sistema de blog utiliza un pipeline de build custom:

1. *Contenido*: Archivos Markdown en =content/blog/[slug]/= con frontmatter YAML
2. *Generación*: =scripts/build-blog.js= parsea directorios y genera =src/data/blogPosts.ts=
3. *Multilenguaje*: Detecta archivos =index.es.md= e =index.en.md= para cada post
4. *Imágenes*: Copia automáticamente imágenes del directorio del post a =public/blog-images/[slug]/=
5. *Renderizado*: ReactMarkdown con plugins para:
   - GFM (GitHub Flavored Markdown)
   - Math (KaTeX para fórmulas)
   - Syntax highlighting (react-syntax-highlighter)
   - Custom image handling

** Pipeline de Build

*** Build Script (=npm run build=)
#+BEGIN_SRC bash
node scripts/build-blog.js && vite build
#+END_SRC

1. Ejecuta =build-blog.js= para generar =blogPosts.ts=
2. Vite compila el proyecto a =build/=
3. Optimiza imágenes con =vite-plugin-image-optimizer=
4. Code splitting: chunks separados para =vendor= (react, react-dom) y =ui= (lucide, recharts)
5. Minificación con esbuild
6. Sourcemaps deshabilitados en producción (seguridad)

*** Contenedor Docker
=Containerfile= usa multi-stage build:

1. *Stage builder*: Node 20 Alpine, instala dependencias, ejecuta build
2. *Stage runtime*: Nginx Alpine sirve archivos estáticos desde =build/=
3. Configuración custom de Nginx en =container/nginx.conf=

** Seguridad

*** Headers HTTP
Vite configura headers de seguridad en modo desarrollo:
- =X-Content-Type-Options: nosniff=
- =X-Frame-Options: DENY=
- =X-XSS-Protection: 1; mode=block=

*** Protección en Build
- Sourcemaps deshabilitados en producción
- Dependencias auditadas con =npm audit=
- Aliasing de packages en Vite para evitar conflictos de versiones

** Configuración de Vite

=vite.config.ts= incluye:
- Plugin React SWC para fast refresh
- Image optimizer con calidad 80% para PNG, JPEG, WebP, AVIF
- Alias =@= pointing to =./src=
- Code splitting manual (vendor, ui)
- Servidor en puerto 3000 con headers de seguridad
